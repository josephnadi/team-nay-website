# Contributing Guidelines

## Git Workflow
- The `main` branch contains stable code
- New features are developed in feature branches
- Changes are merged using Pull Requests

## Branch Naming Convention
- feature/contact-page
- feature/services-page
- feature/about-page
- feature/contact-page2:(Edited certain important information)

## Commit Message Guidelines
- Use clear, descriptive commit messages that would help other users and developers understand easily.
- Example: "Add contact form validation"

## Code Review Process
- All pull requests require review
- At least one approval is needed before merging
- Comments are also advised to be given in areas which needs improvements
